# Grid Implementation Documentation

## Overview

A 3D grid has been added to the viewport, similar to those found in professional 3D software (Blender, Maya, etc.). The grid is rendered on the XZ plane (horizontal plane) at Y=0, providing visual reference for positioning and scale.

## Implementation Details

### Grid Generation

The grid is generated by creating lines along two axes:
1. **X-axis lines**: Lines parallel to the X-axis (extending along Z)
2. **Z-axis lines**: Lines parallel to the Z-axis (extending along X)

All lines are rendered at Y=0 (the horizontal plane).

### Line Types

The grid has three types of lines with different colors:

1. **Center Lines** (Red by default):
   - The line at X=0 (center Z-axis line)
   - The line at Z=0 (center X-axis line)
   - These help identify the origin

2. **Major Lines** (Gray by default):
   - Every 10th grid line
   - Thicker/more prominent than minor lines
   - Help with larger scale measurements

3. **Minor Lines** (Dark gray by default):
   - All other grid lines
   - Provide fine detail for precise positioning

### Rendering Order

The grid is rendered in three passes:
1. **Pass 1**: Render all lines with minor line color
2. **Pass 2**: Overlay major lines with major line color
3. **Pass 3**: Overlay center lines with center line color

This ensures proper visual hierarchy.

### Depth Testing

The grid uses depth testing, so:
- Models appear above the grid
- The grid appears behind models
- Proper 3D depth perception is maintained

## Configuration

The grid can be configured via the following methods:

```cpp
grid.setSize(20.0f);           // Total size (extends from -size/2 to +size/2)
grid.setSpacing(1.0f);         // Distance between grid lines
grid.setMajorLineColor(glm::vec3(0.5f, 0.5f, 0.5f));   // Gray
grid.setMinorLineColor(glm::vec3(0.3f, 0.3f, 0.3f));   // Dark gray
grid.setCenterLineColor(glm::vec3(0.8f, 0.2f, 0.2f));   // Red
grid.setEnabled(true);          // Show/hide grid
```

## Default Settings

- **Size**: 20 units (extends from -10 to +10 in both X and Z)
- **Spacing**: 1 unit between lines
- **Major Line Color**: (0.5, 0.5, 0.5) - Gray
- **Minor Line Color**: (0.3, 0.3, 0.3) - Dark gray
- **Center Line Color**: (0.8, 0.2, 0.2) - Red
- **Enabled**: true

## Integration Points

### Initialization
- Grid is created and initialized after OpenGL context is ready
- Located in `main()` function after shader initialization
- Must be called after `gladLoadGLLoader()` succeeds

### Rendering
- Grid is rendered before models (so models appear on top)
- Located in the main rendering loop
- Uses the same view and projection matrices as models

### Cleanup
- Grid cleanup is called on program exit
- Located in cleanup section before `glfwTerminate()`
- Properly releases OpenGL resources

## Potential Issues and Solutions

### Issue: Grid Not Visible
**Possible Causes**:
- Grid is behind camera
- Grid is disabled (`grid.setEnabled(false)`)
- Grid initialization failed

**Solution**:
- Check camera position (grid is at Y=0)
- Verify `grid.init()` was called successfully
- Check console for initialization messages

### Issue: Grid Appears Above Models
**Possible Causes**:
- Depth testing not enabled
- Grid rendered after models

**Solution**:
- Ensure `glEnable(GL_DEPTH_TEST)` is called
- Verify grid is rendered before models in the loop

### Issue: Grid Lines Too Thick/Thin
**Possible Causes**:
- Line width not set (OpenGL default is 1.0)
- Grid spacing too small/large

**Solution**:
- Adjust grid spacing: `grid.setSpacing(2.0f)` for thicker appearance
- Note: Line width is not currently configurable (uses OpenGL default)

### Issue: Grid Colors Not Visible
**Possible Causes**:
- Colors too dark for background
- Shader not setting color correctly

**Solution**:
- Adjust colors: `grid.setMajorLineColor(glm::vec3(1.0f, 1.0f, 1.0f))` for white
- Check shader compilation errors in console
